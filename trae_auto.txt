RadioDroid 极致简化版修改指南（仅保留收藏播放 + 大屏电视遥控器）

本指南将 xueshi86 魔改版 RadioDroid 简化为一个 纯收藏电台播放器，完全移除在线搜索、本地数据库、分类浏览等复杂功能，仅保留：

· ✅ 从 M3U 文件导入/导出收藏电台
· ✅ 播放器（大屏界面 + 元数据显示）
· ✅ 遥控器焦点导航（上下键切换电台，确认键播放/暂停）
· ✅ 极简的电台列表
· ✅ 主题/语言/播放等必要设置

目标：像 Transistor 一样轻量，但拥有更适配电视的大屏播放界面和更完善的遥控器支持。

修改 ActivityMain.java

1. 移除所有与数据库、服务器、Tabs 相关的代码

· 删除 setupStartUpFragment() 中的复杂逻辑
· 删除 onNavigationItemSelected 中的多处分支（只保留 nav_item_starred 和 nav_item_settings）
· 删除广播接收器中 DATABASE_UPDATED 相关处理

2. 修改 onCreate 末尾，直接显示收藏列表

```java
@Override
protected void onCreate(Bundle savedInstanceState) {
    // ... 原有初始化（主题、工具栏、播放器等）...
```

🎛️ 修改 FragmentStarred.java（收藏列表）

当前 FragmentStarred 已经实现了从 FavouriteManager 读取列表、支持拖拽排序和滑动删除。我们需要确保：

· 列表项点击时播放电台（调用 Utils.showPlaySelection 或直接播放）
· 支持遥控器上下键移动焦点，并自动滚动
· 支持长按弹出菜单（删除、播放等）

添加遥控器焦点管理（参考设计文档中的 FocusManager）

可以在 FragmentStarred 的 onViewCreated 中设置：

```java
private FocusManager focusManager;

@Override
public void onViewCreated(@NonNull View view, @Nullable Bundle savedInstanceState) {
    super.onViewCreated(view, savedInstanceState);
    // 确保 RecyclerView 有焦点
    rvStations.setFocusable(true);
    rvStations.setFocusableInTouchMode(true);
    rvStations.requestFocus();

    focusManager = new FocusManager(rvStations);
    focusManager.initializeFocus(0);
}
```

并在 onResume 中恢复焦点位置。

🔧 简化 PlayerServiceUtil 和 PlayerService

移除所有对 RadioStationRepository 的引用。播放逻辑仅基于 DataRadioStation 对象（包含 StationUuid, Name, StreamUrl 等）。PlayerService 中原本通过 repository 获取电台的逻辑应改为从 FavouriteManager 或传入的 station 获取。

具体修改：

· 在 PlayerService.playCurrentStation() 中，不再从数据库读取，直接使用当前 currentStation
· skipToNext/Previous 从 FavouriteManager 获取下一个电台（循环）
· 删除 MediaSessionCallback 中对 repository 的依赖

🧪 测试要点

1. 启动：直接显示收藏列表（若无收藏则显示空状态和导入按钮）
2. 导入 M3U：从文件选择器导入，列表正确显示

5. 导出：将当前收藏导出为 M3U 文件
6. 删除：滑动或长按删除电台，支持撤销
7. 排序：拖拽调整顺序（可选，保留）
8. 设置：主题切换生效，睡眠定时器工作
